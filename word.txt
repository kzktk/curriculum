【ITとは？】
IT
→インフォメーションテクノロジー。
ソフトウェアを使って情報を加工保存伝達する技術。

【OS】
OS
→オペレーティングシステム。
システム全体を管理するソフトウェアの一つ。
Windows
→米マイクロソフト社から出してるOSのシリーズ名。
macOS (Mac OS X)
→米アップル社から出してるOSのシリーズ名。
CPの装置制御やストレージの管理を行いAPPの実行基盤。
UNIX（ユニックス）
→AT&T社ベル研究所で開発が始まったOSで
その流れを汲むOSの総称。
UNIX認証を得ているほとんどは「商用UNIX」
UNIX風のOSは「UNIX系OS」という。
Linux（リナックス）
→世界で最も普及しているUNIX系OSの一種。
プログラムが公開されていて誰でも自由に使ったり
いじったりする事ができる。

【言語・環境】
HTML
→論理構造や表示の仕方などを書ける。
大半のWEBブラウザはこれ使ってる。
文字の入力。
CSS
→配置や見栄えの記述をする時使う。
HTML文書の見た目をいじれる。
サイズやフォントの設定。
MySQL（マイエスキューアール）
→リレーショナルデータベース（一つのデータに関係するデータを格納する）の管理システム（RDBMS：リレーショナルデータベース管理システム）の一つで最も普及している。
PHP
→OSやAPPの動作や機能などをつける時に使うスクリプト言語
（動きをつける系のやつ）の一種。
WEBに特化している為組み込み系やAIには弱い。
JavaScript
→WEBページに組み込まれたプログラムをブラウザ上で
実行する時に使うスクリプト言語の一種。
ITサービス全般をカバーできる。（WEB、APP、ロボ、AI等）
jQuery（ジェークエリー）
→JavaScriptライブラリの一つ。
ブラウザの種類によらずシンプルなコードで要素や演出効果
アニメーションなどが追加できる。
Apache（アパッチ）
→WEBサーバ（HTTPサーバ）ソフトウェアの一つ。
いろんなOSで使えて最適化されたプログラムを使える。
Bootstrap（ブートストラップ）
→電源を入れてからOSが使えるようになるまでに
自動的に行われる一連の処理。
ブートと略される。

【エンジニアなど】
SE
→システムエンジニア。
ソフトウェアの開発に携わる技術者のうちプログラミング
以外の業務を担当する人のこと。
PG
→プログラマ。
言語を使ってコンピュータプログラムを作る人。
コーディングをする人。
Webデザイナー
→WEBページのデザインを設計する人。

【ユーザーインターフェース】
CUI
→キャラクターユーザーインターフェース。
文字だけで表現されているUIのこと。
コーディングを行う画面のこと。

GUI
→グラフィカルユーザーインターフェース
画像とかアイコンで表現されてるUIのこと。
アプリとかはそれをタップするだけで予めセットされている
プログラムが作動するようなもの。

【ファイルとディレクトリ】
ファイル
→データとかをまとめるもの。
ディレクトリ
→フォルダの別称。
ルート・ディレクトリ
→装置やシステムの一番上にあるフォルダ。
SOL２とかのこと？　
カレント・ディレクトリ
→作業フォルダのこと。現在の位置。　
パス
→ドライブの上に出てきてたやつ。
絶対パス
→ルートディレクトリから省略なく全部記載されているパス。
目的地の住所を教える。
相対パス
→カレントディレクトリ（現在地）からの道筋がわかるパス。
現在地から目的地までの道順を教えてくれる。

【各種プロトコル】予め決まってる共通のルールのこと。
HTTP
→HTML等のコンテンツ送受信の仕組み。
FTP
→ファイル送受信の仕組み。
HTTPS
→HTTPとほぼ一緒だが通信内容を暗号化している為安心。
SSL
→ネット上の通信を暗号化する仕組み
SSH
→ネット上から暗号化して遠隔操作する仕組み
ポート、ポート番号
→動作するソフトウェアのどれと繋げるかを指定する番号。
HTTP、FTP、SSHの各ポート番号
→TCP 80、TCP20.21、TCP22
ファイアウォール
→外部からのウイルスや攻撃から守ってくれるもの。
ソフトウェアや機器、システムのこと。
TCP
→安全性重視で通信するプロトコル
速度は遅い。

【IPアドレス】
IPアドレス
→ネット上に接続した機器が持つ番号のこと。
グローバルIPアドレス
→ネット上に接続する機器が使用できるIPアドレス。
重複厳禁。
プライベートIPアドレス
→組織内のネットでのみ使えるIPアドレス。
オフィス内や自宅とか。

【ドメイン】
ドメイン
→ネット上の住所みたいなこと。
http://www.以降がドメイン名。
DNS
→ドメインネームシステム。
ネット上のドメイン名を管理するシステム。
VPN
→バーチャルプライベートネットワーク。
データの拠点間接続などに使われる。
ブリッジ
→宛先情報を参照して中継の可否を判断する機能。橋渡し。
プロキシサーバー
→内部ネットワークとインターネットの間にあり
内部から外部へアクセス要求をすると代理（proxy）してくれる。

【電子メール】
SMTP
→simple mail transfer protocol
メール送信する際の仕組み。
POP
→post office protocol
メール受信する仕組み。
メールはダウンロードしてサーバーから消す。
IMAP
→internet message access protocol
メール受信する仕組み。
メールをコピーしてサーバーにはメールを残しておく。

【セッションとクッキー】
セッション
→WEBページ上で操作した履歴。
クッキー
→ページを訪れたユーザー情報を一時的に保存する仕組み
キャッシュ
→見たWEBページの情報を一時的に保存する機能。

【DB（データベース）について】
チューニング
→設定や構成を調整する事
スロークエリ
→遅いSQLのこと
インデックス
→同じものが並んでるところに区別できるようにつけるもの、識別番号

【テストについて】
フロー
→処理とかの流れのこと
単体テスト
→一つを対象に行うテスト
結合テスト
→複数を組み合わせて行うテスト

【チケット管理ツール】
チケット管理ツール
→タスク管理するもの。各種タスクの発行を行いそれを依頼することもできる。
タスク
→やること。ソフトウェアの実行単位のこと。

【バージョン管理】
Git
→複数人が編集履歴を統一して管理できるシステム
SourceTree
→gitを操作するソフト
SVN　subversion
→オープンソースの集中型バージョン管理システム
コンフリクト
→同時に同じものを利用しようとして動作が不安定になること
デグレ
→直したものがまた悪くなること
マージツール
→異なる編集者が同じものに変更を加えた時に自動的に統合するシステム

【その他】
ロジック
→手順、流れ
デバッグ
→バグを見つけて取り除くこと
開発手法(ウォーターフォール開発、アジャイル開発)
→ウォーターフォールは各工程に分けて順に段階を踏む方法で
前の工程には戻らない
アジャイルは実装とテストをこまめに繰り返す方法
アルゴリズム
→問題解決に至るやり方や考え方のこと
フローチャート図
→各ステップを図にしたもの
設計書(外部設計、内部設計)
→外部設計はシステムの仕様を決定する。内部設計は外部設計の結果を
実際にプログラミングできるようにシステム内部に特化したもの。
要件定義(要求仕様書)
→開発工程の前段階具体的にどのように進めるか決めるもの。
システム開発におけるテスト(単体テスト、結合テスト、総合テスト)
→単体は機能一つに対して、結合は２つ以上の機能に対して
総合は全体の機能に対して行う。
テスト仕様書(どのようなもの、項目)
→クライアントからのヒアリングをもとに作った要件定義書通りに
機能するかどうかのポイントをまとめたもの。
項目は実施した環境、テスト内容、操作手順、実行結果
テスト項目識別のための番号、年月日、担当者
バリデーション
→対象が使用や文法に対して適切に記述されているか否かを
検証すること
FW（フレームワーク）
→アプリ開発の際に土台として機能させるソフト。枠組みや
骨組みなどの意味がある。
PHPのFW（フレームワーク）の種類と特徴
→Laravel 学種コストが低く人気が最も高いFMで初心者でも綺麗にかける
CakePHP 日本で最も普及していたものでコスト低、軽量で高速開発が可能
CodeIgniter 歴史が古く動作が軽量、高速。コーディング規約が
ひっかう的ゆるく自由度が高い。
Ajax エイジャックス
→WEB内で非同期通信を行いながらインターフェイスの構築を行う手法
API
→アプリに別のプログラムを追加するためのもの。
LAMP　ランプ
→Linux,Apache,MySQL,PHPの頭文字をとったオープンソースの
ソフトウェア群
クラウドサーバー、レンタルサーバー、VPSの違い
→クラウドは使いたい時に使える、レンタルは１台の物理サーバーを
共有してつかう。VPSはレンタルとほぼ一緒だが、仮想的に
複数のサーバーを構築するため、自由度は高い。
AWS
→アマゾンが提供するクラウドコンピューティングサービスの総称
プラットフォーム
→基板や土台、環境を意味する言葉
カバレッジ
→カバーしている範囲をさす
エビデンス
→証拠
コンパイル
→ソースコードをコンピュータ上で実行可能な形式に変更すること。
バッチ処理
→一定量のデータを集めて一括処理する方法
デプロイ(意味と方法、FTPソフトの種類)
→実際の運用環境に配置すること。実行ファイルを作って（ビルド）
サーバーにアップする。
種類はFFFTP、WinSCP、CarotDAV等
データベース接続クライアント(説明、代表的なもの)
→データベース管理システムや関係データベース管理システム上の
データを容易に扱うためのソフトウェア
代表的なものはDBeaver、A5：SQL Mk-２
Gitホスティングサービス(説明、代表的なもの)
→Gitによるクラウドベースのバージョン管理サービス
代表的なものはGitHub、Bitbucket GitLab
チケット管理ツール(説明、代表的なもの)
→タスク管理できるツール。
代表的なものはWrike、Redmine、Backlog
DBのJOINとは
→複数を合体させて１つのテーブルにすること
SEO
→検索エンジン最適化の略称。
優良リンクの収集（外部施策）と適正に検索エンジンにページ内容を
理解、評価されるように技術的に最適化（内部施策）することで自サイトを上位に表示させるようにすること。
